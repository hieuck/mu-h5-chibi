# Game Concept: MU Chibi Squad

*Created: 2026-06-20*
*Status: Draft*

---

## Elevator Pitch

> It's an idle ARPG where you build a squad of chibi MU-style heroes, auto-farm monsters for legendary gear sets and wings, and strategically optimize team compositions — all while progressing even when you're offline. Play anywhere, on web, mobile, or desktop.

## Core Fantasy

You are a guild master leading a squad of chibi warriors through the continent of MU. You don't need to click 12 hours a day — you build the perfect team, equip them with legendary sets, and watch them grow stronger. Every morning you wake up to new loot, every week you push past another boss, every month your squad resets and reaches new heights. The fantasy is **progression without slavery**: all the satisfaction of MU's gear chase and power growth, none of the grind.

## Unique Hook

"It's like MU Online, AND ALSO your team keeps fighting for you when you're offline — you focus on strategy, not grinding."

## Player Experience Analysis (MDA Framework)

### Target Aesthetics

| Aesthetic | Priority | How We Deliver It |
| ---- | ---- | ---- |
| **Challenge** | 1 | Boss/dungeon puzzles, team optimization puzzles, reset strategy |
| **Fantasy** | 2 | MU world aesthetics, wings, legendary sets, class fantasy chibi-fied |
| **Submission** | 3 | Auto-farm loop, satisfying loot notifications, low-pressure progression |
| **Expression** | 4 | Build diversity, team composition, cosmetic wings/outfits |
| **Discovery** | 5 | Hidden set bonuses, team synergies, secret boss mechanics |
| **Fellowship** | 6 | Guild contributions, leaderboards, trading (future), co-op boss (future) |
| **Sensation** | 7 | Flashy skill effects, satisfying loot sounds, wing flutters |
| **Narrative** | 8 | Light lore — world setting, class backstories |

### Key Dynamics

- Players will experiment with team compositions to discover synergies
- Players will theory-craft optimal gear sets for each game mode
- Players will share builds and strategies with the community
- Players will develop emotional attachment to their squad members

### Core Mechanics

1. **Auto-Farm Combat**: Squad automatically fights monsters on the current map; player can override skills or set tactics
2. **Team Building System**: 3-4 character slots, class switching, formation, AI behavior tuning
3. **Equipment & Set System**: MU-style gear (helm/armor/pants/gloves/boots/weapon/wings) with set bonuses, socket gems, enhancement
4. **Reset Progression**: Classic MU reset — reset level while keeping gear, permanent stat bonuses compound
5. **Manual Boss Raids**: 10% manual gameplay — player-controlled skill timing for boss/dungeon encounters

## Player Motivation Profile

### Primary Psychological Needs Served

| Need | How This Game Satisfies It | Strength |
| ---- | ---- | ---- |
| **Autonomy** | Choose team composition, gear builds, skill loadouts, progression path | Core |
| **Competence** | Mastery through team optimization, solving boss mechanics, efficient progression | Core |
| **Relatedness** | Guild contributions, leaderboard competition, sharing builds | Supporting |

### Player Type Appeal (Bartle Taxonomy)

- [x] **Achievers** — Goal completion, collection of full sets, reset milestones, progression efficiency
- [x] **Explorers** — Discovering team synergies, hidden item effects, optimal builds
- [x] **Socializers** — Guild community, sharing progress, competitive leaderboards
- [ ] **Killers/Competitors** — Not primary; leaderboards suffice for light competition

### Flow State Design

- **Onboarding curve**: First 10 minutes introduce ONE hero, basic auto-farm, equip first item — immediate satisfaction
- **Difficulty scaling**: Maps require progressively better gear; bosses test team synergy, not APM
- **Feedback clarity**: Loot sparks, numbers pop, gear comparison tooltips, set bonus highlights
- **Recovery from failure**: No permadeath; retry boss with adjusted strategy, gear check warnings

## Core Loop

### Moment-to-Moment (30 seconds)
Open game → check auto-farm results (loot sparkles) → quick-compare items → equip upgrades → change map or restart auto → close. Satisfying loot ping, visible power gain, zero friction.

### Short-Term (5-15 minutes)
Deep inventory management: compare gear, enhance items, socket gems, adjust team formation, re-spec skills. Push one boss attempt manually. Check daily missions.

### Session-Level (30-120 minutes)
Auto-farm session during work/errands → return to significant loot pool → gear overhaul → try new team comp → push progression chain: 2-3 new maps → elite boss → upgrade cycle → set next auto target.

### Long-Term Progression
- **Daily**: Check loot, quick missions, guild contributions
- **Weekly**: Boss reset, weekly dungeons, arena ranking
- **Monthly**: Major content update, new class release, seasonal event
- **Perpetual**: Reset cycle (level 1→400 → reset → stronger), wing evolution stage 1→2→3, set collection completion

### Retention Hooks
- **Curiosity**: "What if I try Elf + Dark Wizard combo? What set drops from map 5 boss?"
- **Investment**: "My Dark Knight is 3 pieces away from full Ancient set — can't stop now"
- **Social**: Guild weekly boss ranking, friends' progress comparison
- **Mastery**: "Can I beat Hell dungeon with this off-meta build?"

## Game Pillars

### Pillar 1: Strategy over APM
Players win through smart team building and gear optimization, not click speed or reaction time.

*Design test*: If a feature rewards 12-hours-a-day grinding over clever strategy, this pillar says we nerf the grind and buff the strategy.

### Pillar 2: MU Fantasy, Chibi Style
Sets, wings, reset — the soul of MU Online preserved, reimagined in adorable 4-head chibi proportions.

*Design test*: If a system doesn't make the player feel like their character is becoming a MU legend, it doesn't belong.

### Pillar 3: Depth from Systems, Not Grind
Strategic complexity comes from team synergies, gear interactions, and build variety — not from time spent clicking monsters.

*Design test*: If auto-farm were removed, the remaining strategic layer should still be compelling enough to retain players.

### Pillar 4: Cross-Platform First
Every feature is designed for the lowest common denominator (mobile touch), then enhanced for desktop.

*Design test*: If a feature can't work on a phone screen, it needs redesigning or it's relegated to post-launch.

### Anti-Pillars

- **NOT a grindfest**: We will NOT require 8+ hours daily play. Auto-farm fills the gap, not replaces skill.
- **NOT P2W**: We will NOT sell power. Cosmetics, convenience, and accelerated (but capped) progression only.
- **NOT real-time PVP**: We will NOT build competitive real-time PVP. Async ranking and leaderboards only.
- **NOT an MU clone**: We will NOT copy MU exactly. We use MU as inspiration, not a blueprint.

## Visual Identity Anchor

### Visual Direction: "Chibi Fantasy Premium"
Chibi proportions (4-5 heads), vibrant high-saturation palette, cel-shaded 2D sprites with glow effects for equipment. Detailed equipment visualization — every gear piece visibly changes the character model.

### Visual Rule
Players must be able to identify another player's power level at a glance by their character's silhouette and glow.

### Supporting Principles
- **Gear visibility**: Every equipment slot changes the sprite visually — no invisible stats
- **Wings as status symbols**: Wing size and particle effects directly communicate progression stage
- **Juicy feedback**: Hits, crits, loot drops, and level-ups all have screen-following particle/VFX responses

### Color Philosophy
Warm fantasy palette (gold, crimson, deep blue) with equipment tier colors: Normal (white) → Magic (blue) → Rare (yellow) → Set (green) → Ancient (red) → Mythic (purple glow).

## Technical Considerations

| Consideration | Assessment |
| ---- | ---- |
| **Recommended Engine** | Phaser 3 + TypeScript — best H5 game engine, huge community, isometric tilemap support, WebGL rendering |
| **Key Technical Challenges** | Real-time auto-combat simulation offline, cross-platform save sync, isometric rendering with z-ordering for team of 4 |
| **Art Style** | 2D isometric chibi sprites with 3D-like lighting (sprite swapping for equipment layers) |
| **Art Pipeline Complexity** | Medium — custom 2D sprite pipeline for character + equipment layering |
| **Audio Needs** | Moderate — BGM per map, SFX per skill, loot sounds, UI feedback |
| **Networking** | Minimal — async only: cloud save, leaderboard sync, guild data. No real-time multiplayer |
| **Content Volume (MVP)** | 5 maps, 5 classes, ~50 equipment sets, 3 wing tiers, 10 bosses, ~100 items |
| **Procedural Systems** | None in MVP — hand-crafted maps and loot tables (procedural dungeons = post-launch) |

## Risks and Open Questions

### Design Risks
- Idle + team management may feel shallow if gear/team synergy lacks depth
- MU fans may reject "chibi" style as too cute for the MU universe
- 90% auto may make the game feel like a "spreadsheet simulator" without enough visual payoff

### Technical Risks
- Phaser 3 isometric rendering for 4 simultaneous characters + effects may hit performance on low-end mobile
- Offline progression simulation must be lightweight and cheat-proof
- Cross-platform save sync requires robust cloud infrastructure

### Market Risks
- Idle RPG market is saturated (AFK Arena, etc.)
- MU Online fanbase is aging (average age ~30+) — may not convert to mobile
- Chibi style may target too young an audience for the MU theme

### Scope Risks
- Equipment sprite layering (every slot visible) creates exponential art cost
- 5 classes × gear × wings = asset explosion for solo dev
- Cross-platform testing matrix (web/mobile/Steam) may triple QA time

### Open Questions
- Does the target audience prefer classic isometric 2D or are they open to 2.5D with 3D models?
- Should manual boss fights be real-time (click to dodge/ult) or tactical (pause & command)?
- Does offline progression need its own sim environment (like AFK Arena) or can it simplified?

## MVP Definition

**Core hypothesis**: Players find the gear chase + team optimization loop compelling even at 90% auto-play.

**Required for MVP**:
1. 5 playable classes: Dark Knight, Dark Wizard, Elf, Summoner, Magic Gladiator
2. 5 auto-farm maps with scaling difficulty
3. Full equipment system: 8 slots, set bonuses, enhancement, socket gems
4. Wing system: 3 tiers with visual progression
5. Reset system: up to 5 resets with permanent stat gains
6. 3 manual boss encounters with unique mechanics
7. Basic auto-farm AI with configurable tactics
8. Cloud save (player account system)

**Explicitly NOT in MVP** (defer to later):
- Guild system
- Castle siege
- Trading / auction house
- PVP arena
- Procedural dungeons
- Seasonal events
- Social features (chat, friend list)

### Scope Tiers

| Tier | Content | Features | Timeline |
| ---- | ---- | ---- | ---- |
| **MVP** | 5 maps, 5 classes, gear + wings + reset | Auto-farm, 3 bosses, basic upgrade, cloud save | 9-12 months |
| **Alpha** | 10 maps, 7 classes, ancient sets | Full skill tree, gem socket, guild scaffold | 12-16 months |
| **Beta** | 15 maps, complete skill system | Castle siege, trading, guild wars | 16-20 months |
| **Full Vision** | 20+ maps, all classes, endgame raids | Seasonal events, PVP arena, Steam release | 20-24 months |
